# /draft-brief {사건폴더명}

## 전제 확인 (없으면 안내 후 중단)

- `사건컨텍스트.json` + `리서치/쟁점별_법리.md` 존재.
- `입력/`에 상대방 서면 텍스트(예: 상대방_답변서.md) 존재. 없으면 상대 서면을
  텍스트로 변환해 `입력/`에 넣어 달라고 안내.
- 사건번호: `사건컨텍스트.사건.사건번호`가 null이면 사용자에게 질문(소 제기 후
  부여된 번호). 답을 받으면 사건컨텍스트에 기록.
- 리서치에 "예상 항변과 재반박" 절이 없으면(경량 모드 산출) 상대 주장 반박
  재료가 준비되지 않은 상태다 — /legal-research 표준 모드 보강을 먼저 안내.

## 상대 주장 분석

1. 상대 서면을 읽고 주장을 단위별로 분해해 표로 정리:
   | 상대 주장 | 대응 분류(인정/부인/항변) | 우리 반박 근거(사실+호증) | 관련 쟁점 id |
2. `사건컨텍스트.쟁점[].상대주장`을 갱신. 상대 서면에서 **새 쟁점**이 나오면
   `쟁점[]`에 추가하되, 그 쟁점의 법리가 리서치에 없으면 **작성을 중단하고
   /legal-research 보강을 안내한다** (즉석 법리 작성 금지).
3. 분해 표를 사용자에게 보여 확인받은 뒤 본문 작성으로 진행.

## 본문 구성 — 절(번호/제목/문단[]) 배열

1. "이 사건의 쟁점" — 쟁점 목록을 ①②③으로 정리.
2.~ 쟁점별 절: 피고(상대) 주장 요지 → 반박(사실관계 근거 + 호증 표시) →
   법리(리서치 발췌·출처 핀 인용). 상대방 증거에 대한 의견도 해당 절에 기재.
   핀 블록은 통째로 복사한다 — `[행위시 확인: …]` 부기 포함(인용규칙 §7),
   `[cite_check: …]` 라인은 서면 제외.
- **소장 중복 기재 금지**: 소장에 이미 쓴 사실·법리는 "소장 기재와 같으므로
  원용합니다" 수준으로 처리.
- 마지막 절은 결론. 확인 필요 사항은 본문에 넣지 않는다 — context의
  `확인필요목록` 배열로 채우면 템플릿이 문서 맨 뒤 **별첨 페이지**(제출 전
  삭제 대상)로 렌더한다(소장과 동일 구조).
- **부존재·부작위 단정 가드**: 입력 자료가 명시적으로 확인해주지 않는
  부존재·부작위는 단정하지 않는다("…한 사실이 전혀 없습니다" 금지).
  완화 어투("…한 사실이 확인되지 않습니다")로 쓰고 `[변호사 확인 필요]`
  표기 + 사건컨텍스트 `확인필요`에 적재한다. 의뢰인이 입력에서 명시적으로
  진술한 부존재는 단정 가능(근거 표기 필수).

## 렌더 전 자체 점검 (필수 — check_projection 실행 직전)

1. **핀 점검**: context JSON의 조문·판례 언급 문장 전수 — 각 문장에 출처
   핀이 있고, 핀·발췌가 리서치의 해당 항목과 일치하는가(`[행위시 확인]`
   부기 포함 — 인용규칙 §7).
2. **사실 근거 점검**: 사실 주장 전수(당사자 배경 진술 포함) — 각각
   사건컨텍스트.사실관계(근거 포함) 또는 당사자 필드에 대응하는가.
3. **부존재 단정 스캔**: 부존재·부작위 단정이 3분법 가드를 지키는가.

세 항목을 통과할 때까지 context JSON을 수정한 뒤 렌더로 진행한다. 결과는
대화창 보고에 1줄로 기재("자체 점검: 핀 n/n · 사실근거 n/n · 부존재단정
위반 0") — **파일로는 산출하지 않는다.** /verify-citations 독립성 불변:
검증자는 자체 점검 결과를 신뢰하지 않고 전수 재검한다.

## 렌더

1. `cases/{사건폴더명}/산출물/context_준비서면.json` 작성 — 키 구조는
   `templates/context_준비서면_예시.json`과 동일.
   입증방법은 이번 서면에서 **새로 내는** 증거만(이미 소장에서 낸 갑호증 제외).
2. `python "templates/check_projection.py" 준비서면 "cases/{사건폴더명}/산출물/context_준비서면.json"` → `OK`
3. `python "templates/render_서면.py" "templates/준비서면_템플릿_docxtpl.docx" "cases/{사건폴더명}/산출물/context_준비서면.json" "cases/{사건폴더명}/산출물/준비서면_초안.docx"`
4. 보고: 초안 경로(`cases/{사건폴더명}/산출물/준비서면_초안.docx`) + 쟁점별 대응 요약 + "/verify-citations {사건폴더명} 준비서면
   통과 + 변호사 검수 전 제출 금지" 고지.
