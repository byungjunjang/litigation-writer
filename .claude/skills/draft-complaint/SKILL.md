# /draft-complaint {사건폴더명}

## 전제 확인 (하나라도 없으면 해당 스킬 먼저 하라고 안내 후 중단)

- `cases/{사건폴더명}/사건컨텍스트.json` 존재 + `쟁점[]` 비어 있지 않음 → /case-intake
- `cases/{사건폴더명}/리서치/쟁점별_법리.md` 존재 → /legal-research

## 요건 점검 (작성 전)

풀 대조는 하지 않는다 — /case-intake가 유일한 풀 대조 지점이다. 두 가지만 한다:

1. `사건컨텍스트.확인필요`의 `[요건]` 접두 항목 중 미해소분을 점검한다.
   미해소 요건이 있어도 작성은 진행하되(가시화 게이트), 해당 법적 주장 절에
   `[변호사 확인 필요 — 요건사실 ○○ 미충족 가능]`을 표기하고 최종 보고에 명시.
2. intake 이후 청구유형이 추가·변경된 경우에만, 그 유형에 한해
   `rules/요건사실.md` 재대조를 수행한다(결과 적재 방식은 case-intake와 동일).

## 작성

1. **청구취지** — `사건컨텍스트.청구[]`에서 도출:
   - 이행: "피고는 원고에게 …을 지급하라." 금전청구는 지연이자 문구 포함.
   - **지연이자 확정수준**: 기산일·이율 중 ⓐ 리서치에 원문 발췌된 조문으로
     뒷받침되고(적용 대상·요건이 조문 문언상 명확한 경우에 한한다) ⓑ 기산
     기준일이 사건컨텍스트.사실관계에서 도출되는 것은 **금액 미확정과
     무관하게 청구취지에 명시**한다(핀 병기). 예: 임금·퇴직금 → 근로기준법
     제37조·같은 법 시행령 제17조에 따라 "지급사유 발생일부터 14일이 되는
     날의 다음 날부터 다 갚는 날까지 연 20%". ⓐ·ⓑ 미충족 부분(금액, 원문
     미반환 대통령령 이율, 적용 대상이 문언상 불명확한 항목 등)만
     "연 ○○%" + " [변호사 확인 필요 — 적용 이율]"로 남긴다. 확정 반영한
     이율에 대응하는 `확인필요` 항목은 삭제하지 않고 미확정 잔여(적용 범위
     등)로 좁혀 갱신한다.
   - 확인: "…은 무효임을 확인한다." / 형성: 해당 정형 문안.
   - 고정 말미: "소송비용은 피고가 부담한다." + 금전청구 있으면
     "제n항은 가집행할 수 있다." (n = 해당 금전청구 항 번호)
2. **청구원인** — 절(번호/제목/문단[]) 배열:
   - 1. 당사자의 지위 → 2. 사실관계(사건컨텍스트.사실관계 시간순, 각 사실 끝에
     호증 표시 "(갑n)") → 3.~ 쟁점별 법적 주장 → 마지막-1. 결론.
   - "당사자의 지위" 절의 각 사실 진술은 당사자 필드(성명·명칭·대표자 등
     신원 정보) 또는 사실관계(근거 포함)에서만 가져온다. 어느 쪽에도 없는
     배경사실은 쓰지 않는다(필요하면 `확인필요` 적재).
   - 법적 주장 절은 `리서치/쟁점별_법리.md`의 발췌·출처 핀을 그대로 인용해 구성.
     **리서치에 없는 조문·판례가 필요해지면 작성을 중단하고 /legal-research
     보강을 안내한다.**
     핀 블록은 통째로 복사한다 — 출처 핀에 `[행위시 확인: …]` 부기가 붙어
     있으면 부기까지 함께 옮긴다(인용규칙 §7). `[cite_check: …]` 라인은
     리서치 전용이므로 서면에 넣지 않는다.
   - 마지막 절은 결론. 확인 필요 사항은 청구원인에 넣지 않는다 — context의
     `확인필요목록` 배열(`사건컨텍스트.확인필요` 전체, 번호 없이 문장만)로 채우면
     템플릿이 문서 맨 뒤 **별첨 페이지**(제출 전 삭제 대상)로 렌더한다.
   - 출처 핀 예외 없음: 결론 절·확인필요목록을 포함해 조문·판례를
     언급하는 모든 문장에 핀을 병기한다(인용규칙 절대 규칙 2).
3. **입증방법** = 사건컨텍스트.증거, **첨부서류** = 통상 5종(위 입증방법 각 2통,
   법인등기부등본 1통[피고 법인일 때], 영수필확인서 및 영수필통지서 1통,
   송달료납부서 1통, 소장 부본 1통), **작성일** = 오늘, **제출법원** = 사건컨텍스트.
- **부존재·부작위 단정 가드**: 입력 자료가 명시적으로 확인해주지 않는
  부존재·부작위는 단정하지 않는다("…한 사실이 전혀 없습니다" 금지).
  완화 어투("…한 사실이 확인되지 않습니다")로 쓰고 `[변호사 확인 필요]`
  표기 + 사건컨텍스트 `확인필요`에 적재한다. 의뢰인이 입력에서 명시적으로
  진술한 부존재는 단정 가능(근거 표기 필수).

## 절차·비용 산출 (렌더 전 — 대화창 보고 전용)

`rules/절차비용.md` 기준으로 산출해 **대화창 보고에만** 포함한다(근거 조문 핀
병기). 서면 본문·context JSON에는 제출법원 외 추가하지 않는다.

1. **소가 산정** — 금전청구는 청구금액(지연이자 등 부대목적 불산입),
   비재산권 청구(해고무효확인 등)는 인지규칙 제18조의2 기준액, 병합 시 합산·
   다액 기준 규칙 적용. 산정 불가 항목은 `[변호사 확인 필요]`.
2. **인지액 계산** — 구간 산식 적용 과정을 표시(끝수 처리 포함), 전자소송
   금액(10분의 9) 병기.
3. **송달료** — 산식 틀(당사자 수 × 회분 × 단가)과 당사자 수만 제시, 금액은
   `[변호사 확인 필요]`.
4. **관할 후보** — 합의관할 → 피고 보통재판적 → 의무이행지 → 관련재판적
   순서로 후보 법원 2~3개 도출, 각 근거 조문 핀 병기. 최종 선택은
   `[변호사 확인 필요]`.
5. 산출 결과 중 미확정 값은 `사건컨텍스트.확인필요`를 갱신한다.

## 렌더 전 자체 점검 (필수 — check_projection 실행 직전)

1. **핀 점검**: context JSON의 조문·판례 언급 문장 전수 — 각 문장에 출처
   핀이 있고, 핀·발췌가 리서치의 해당 항목과 일치하는가(`[행위시 확인]`
   부기 포함 — 인용규칙 §7).
2. **사실 근거 점검**: 사실 주장 전수(당사자의 지위 절 포함) — 각각
   사건컨텍스트.사실관계(근거 포함) 또는 당사자 필드에 대응하는가.
3. **부존재 단정 스캔**: 부존재·부작위 단정이 3분법 가드를 지키는가.

세 항목을 통과할 때까지 context JSON을 수정한 뒤 렌더로 진행한다. 결과는
대화창 보고에 1줄로 기재("자체 점검: 핀 n/n · 사실근거 n/n · 부존재단정
위반 0") — **파일로는 산출하지 않는다.** /verify-citations 독립성 불변:
검증자는 자체 점검 결과를 신뢰하지 않고 전수 재검한다.

## 렌더

1. `cases/{사건폴더명}/산출물/context_소장.json` 작성 — 키 구조는
   `templates/context_소장_예시.json`과 동일 (스칼라 15개 + 배열 5개,
   `확인필요목록` 포함 — 비어 있으면 "없음" 1항목).
   당사자는 원고[0]·피고[0] 기준(v1 한계).
2. 필수 키 검사 (실패 시 JSON 수정 후 재시도):
   `python "templates/check_projection.py" 소장 "cases/{사건폴더명}/산출물/context_소장.json"` → `OK`
3. 렌더:
   `python "templates/render_서면.py" "templates/소장_템플릿_docxtpl.docx" "cases/{사건폴더명}/산출물/context_소장.json" "cases/{사건폴더명}/산출물/소장_초안.docx"`
4. **보고**: 초안 경로(`cases/{사건폴더명}/산출물/소장_초안.docx`), 청구취지 요약, 확인필요 목록, 그리고
   "/verify-citations {사건폴더명} 소장 통과 + 변호사 검수 전 제출 금지" 고지.
