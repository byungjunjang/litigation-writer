# /case-intake {사건폴더명}

`cases/{사건폴더명}/입력/`의 자료를 구조화해 `cases/{사건폴더명}/사건컨텍스트.json`을
생성·갱신한다. 스키마: `rules/사건컨텍스트.schema.json`. 안전수칙: 루트 CLAUDE.md.

## 절차

1. **폴더 확인.** `cases/{사건폴더명}/`이 없으면 `입력/`, `리서치/`, `산출물/` 하위
   폴더를 만들고, 상담메모 등 텍스트 자료를 `입력/`에 넣어 달라고 안내한 뒤 중단.
   폴더명 규칙: `{연도}_{의뢰인}_{사건요지}` — 구분자는 언더스코어·공백 모두 허용
   (예: `2026_김민재 해고 무효` — 골든 케이스와 동일 형식).
2. **입력 전부 읽기.** `입력/`의 모든 .md/.txt 파일을 Read로 읽는다.
3. **기존 컨텍스트 병합.** `사건컨텍스트.json`이 이미 있으면 그것을 기준으로
   추가·보정만 한다. 기존 값을 임의로 삭제하지 않는다(멱등 갱신).
4. **스키마에 따라 작성:**
   - `사실관계`: 입력에 명시된 사실만, 시간순. 각 항목에 `근거` 필수
     (예: "상담메모.md §2" 또는 "갑2"). 일자는 `YYYY.MM.DD.`, 불명확하면 `YYYY.MM.경`.
     당사자의 지위·배경으로 서면에 진술될 사실(업종·규모·직위·근속 등)도
     입력에 명시된 것에 한해 근거와 함께 등재한다 — 입력 근거가 없으면
     등재하지 않고 `확인필요`로. 이런 배경사실의 일자는 그 사실이 법적
     의미를 갖는 기준 시점(예: 해고 무렵 상시근로자수 → "2025.11.경") 또는
     입력 자료 기준 시점의 `YYYY.MM.경`으로 기재한다.
   - `당사자`: 입력에서 추출. 성명/명칭·주소 등 필수 정보가 없으면 사용자에게
     질문(AskUserQuestion)하고, 끝내 미확보면 빈 값 + `확인필요`에 적재.
   - `증거`: 입력에 언급된 자료를 갑호증 후보로 정리(표시는 갑1부터 순번).
   - `청구`: 의뢰인이 원하는 결과를 이행/확인/형성으로 분류해 청구취지 문안 초안.
     금액은 입력에 있는 수치만 사용, 산정내역 불명이면 `확인필요`에 적재.
   - `확인필요`: 지연이자 이율, 관할(보통재판적·합의관할), 미확보 인적사항 등
     모르는 값 전부.
5. **요건사실 대조.** `청구`의 청구유형을 `rules/요건사실.md`와 대조한다:
   - 해당 유형의 요건사실별로 대응하는 `사실관계` 항목(근거 포함)과 `증거`가
     있는지 매핑한다.
   - 누락 요건은 **"의뢰인 추가 확인 질문" 표**(요건 / 질문 / 필요한 증거)로
     대화창에 출력하고 `확인필요`에 적재한다. 적재 시 `[요건] {청구유형}
     {요건 번호·명} — {내용}` 형식의 접두어를 붙인다(/draft-complaint가 이
     접두어로 미해소 요건을 점검한다 — 이 단계가 유일한 풀 대조 지점).
   - 사건컨텍스트에는 조문·법리를 기록하지 않는다(아래 금지 유지 — 요건사실은
     질문 생성·점검에만 사용).
   - 사전에 없는 청구유형이면 그 사실을 보고하고 통상 절차로 진행한다
     (차단하지 않음). 추후 요건사실 확인 시 사전에 추가.
6. **쟁점 승인.** 쟁점 후보를 도출해 표(id/제목/우리주장)로 제시하고 사용자
   승인·수정을 받은 뒤에만 `쟁점[]`에 기록하고 파일을 저장한다.
   재실행 시 기존 `쟁점[]`과 동일한 쟁점은 승인 절차를 생략한다(멱등 갱신) —
   승인이 필요한 것은 새로 추가·변경되는 쟁점뿐이다.
7. **보고.** 당사자 요약, 사실관계 건수, 요건사실 매핑 결과(충족/누락), 확정 쟁점,
   청구, 확인필요 목록을 출력하고 "검수 후 /legal-research 진행"을 안내.

## 금지

- 입력에 없는 사실·수치·날짜 생성 금지. 추정 필요 항목은 `확인필요`로.
- 조문·판례 인용 금지 — 법리는 /legal-research의 몫.
- 추론·종합으로 만든 진술을 의뢰인 진술로 귀속 금지 — 입력 자료에 '명시된 문장'만 의뢰인 진술로 인용할 수 있다. 입력에 없는 의사·동의 여부는 사실이 아니라 확인필요 항목이다.
