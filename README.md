# 송무서면 생성기 — 소장·준비서면 작성 에이전트

> 상담메모를 넣으면, AI가 법령·판례를 찾아 **소장·준비서면 초안(워드 파일)** 을 만들어 주는 작업공간입니다.
> Claude Code 또는 Codex 채팅창에 한 줄만 입력하면 됩니다. 명령어 암기는 필요 없습니다.

⚠️ **모든 산출물은 "초안"입니다. 변호사가 직접 검수하기 전에는 절대 법원에 제출하지 마세요.**

---

## 이게 뭔가요? (1분 요약)

변호사 1인이 민사사건 서면 작업을 할 때, 단순 반복 작업을 AI에게 맡기는 도구입니다.

**여러분이 하는 일** — 상담메모를 사건 폴더에 넣고, 채팅창에 요청하고, 나온 초안을 검수합니다.
**AI가 하는 일** — 사건 정리, 법령·판례 원문 검색, 초안 작성, 그리고 **모든 인용이 진짜인지 전수 검증**까지.

이 도구가 일반 ChatGPT식 사용과 다른 점:

- **사실을 지어내지 않습니다** — 입력 자료에 없는 날짜·금액·사실은 쓰지 않고, 모르는 값은 `[변호사 확인 필요]`로 표시합니다.
- **인용을 지어내지 않습니다** — 법령·판례는 국가법령정보센터(korean-law-mcp)에서 **실제로 조회되는 원문**만 인용하고, 마지막에 전수 대조 검증을 통과해야 합니다.
- **결과물이 진짜 워드 파일입니다** — 법원 서면 양식이 잡힌 `.docx`로 나와서 바로 열어 고칠 수 있습니다.

## 처음 설치하는 분을 위한 준비 (10분, 최초 1회만)

### 1단계. Claude Code 설치

[Claude Code 공식 안내](https://claude.com/claude-code)를 따라 설치합니다. (Claude 유료 구독 필요)

### 2단계. 이 작업 폴더 준비하기

이 폴더를 GitHub에서 받았다면, 터미널(Windows라면 PowerShell)을 열고:

```bash
git clone https://github.com/<계정명>/<저장소명>.git
cd <저장소명>
```

폴더(압축파일)째 전달받았다면 원하는 위치에 풀어두기만 하면 됩니다.

> `git`이 없다고 나오면 [git-scm.com](https://git-scm.com/)에서 Git을 먼저 설치하세요.

### 3단계. Python 준비물 설치

워드 파일을 만들 때 쓰는 도구 하나만 설치하면 됩니다. 한 번만 실행하세요:

```bash
pip install docxtpl
```

### 4단계. 법령·판례 조회 도구(korean-law-mcp) 설정

이 도구는 법령·판례 원문을 **korean-law-mcp**(국가법령정보센터 Open API를 감싼 MCP
서버)로 그때그때 조회해서, 실제로 존재하는 원문만 인용합니다. 최초 1회만 설정하면 됩니다.

1. **설치** (Node 18 이상 필요):

   ```bash
   npm install -g korean-law-mcp
   korean-law-mcp --version
   ```

2. **API 키 발급** — [open.law.go.kr](https://open.law.go.kr)에서 무료로 발급받습니다.

3. **키 등록** — 발급받은 키를 `LAW_OC` 환경변수로 설정하거나, 작업 폴더 루트에
   `.env` 파일을 만들어 아래처럼 적어둡니다(이 파일은 `.gitignore`에 이미 등록되어 있어
   커밋되지 않습니다):

   ```
   LAW_OC=발급받은키
   ```

   > 키는 어떤 경우에도 커밋하지 마세요.

4. **서버 등록 확인** — MCP 서버 등록은 이 저장소의 `.mcp.json`에 이미 되어 있습니다.
   Claude Code(또는 Codex)를 다시 시작하면 `korean-law` MCP 도구가 자동으로 로드됩니다.

## 쓰는 법 — Claude Code 채팅창에 한 줄

작업 폴더에서 `claude`를 실행한 뒤, 사건 진행 순서대로 한 단계씩 요청합니다.
**각 단계가 끝나면 나온 파일을 변호사가 직접 검수·수정하고, 다음 단계로 넘어갑니다.**

### ① 사건 등록 — `/case-intake`

상담메모(텍스트 파일)를 준비하고:

```
/case-intake 새 사건이야. 의뢰인 홍길동, 임금 미지급 사건. 상담메모는 바탕화면의 메모.txt
```

↓

`cases/2026_홍길동_임금미지급/사건컨텍스트.json` 생성 — 당사자, 사실관계, 쟁점이 정리된 사건 파일입니다.

### ② 법리 리서치 — `/legal-research`

```
/legal-research 홍길동 사건 리서치해줘
```

↓

`리서치/쟁점별_법리.md` 생성 — 쟁점마다 관련 조문·판례의 **원문 발췌**가 정리됩니다.
이후 서면에는 이 파일에 발췌된 것만 인용할 수 있습니다(지어낸 인용 원천 차단).

### ③ 소장 초안 — `/draft-complaint`

```
/draft-complaint 홍길동 사건 소장 써줘
```

↓

`산출물/소장_초안.docx` 생성 — 청구취지·청구원인이 작성된 워드 파일.

### ④ 인용 검증 — `/verify-citations`

```
/verify-citations 홍길동 사건 소장 검증해줘
```

↓

`산출물/검증보고_소장.md` 생성 — 초안의 모든 법령·판례 인용을 원본과 하나하나 대조한 보고서.
**❌가 하나라도 있으면 그 서면은 "제출 금지" 상태**입니다. 고친 뒤 다시 검증하세요.

### ⑤ 상대방 답변서가 오면 — `/draft-brief`

같은 사건 폴더에서:

```
/draft-brief 상대 답변서 왔어. 파일은 입력 폴더에 넣어뒀어
```

↓

`산출물/준비서면_초안.docx` 생성 → 다시 `/verify-citations`로 검증.

### 전체 흐름 한눈에

```
상담메모 → ① /case-intake → 검수 → ② /legal-research → 검수
        → ③ /draft-complaint → ④ /verify-citations → 변호사 최종 검수
상대 답변서 도착 → ⑤ /draft-brief → ④ /verify-citations → 변호사 최종 검수
```

## Codex에서도 쓸 수 있습니다

이 저장소는 Claude Code와 **Codex**(OpenAI의 코딩 에이전트 — CLI/cloud/web)
양쪽에서 동작하도록 구성되어 있습니다.

- **진입점** — Codex는 루트의 [`AGENTS.md`](AGENTS.md)를 읽고 같은 5단계
  절차와 안전수칙(사실 생성 금지, 인용 전수 검증 등)을 그대로 따릅니다.
- **쓰는 법은 동일** — Codex 채팅창에 자연어로 요청하면 됩니다.
  예: `새 사건이야. 의뢰인 홍길동, 임금 미지급. 상담메모는 입력 폴더에 넣어뒀어`
- **절차 정의 위치** — Claude Code용 `.claude/skills/`가 원본이고,
  Codex용 `.codex/skills/`는 거기서 자동 생성되는 미러입니다.
  `.codex/skills/`는 직접 수정하지 마세요.
- **준비물 동일** — `pip install docxtpl` + korean-law-mcp(`LAW_OC` 키 설정,
  4단계 참조). 네트워크가 차단된 Codex 환경에서는 국가법령정보센터 API를 호출할
  수 없어 리서치·검증 단계가 진행되지 않습니다(환경 설정에서 네트워크 허용 필요).

## 폴더 안내

```
.
├── README.md                    ← 지금 보는 파일
├── CLAUDE.md                    ← 에이전트(AI)가 따르는 작업 지침서 (Claude Code)
├── AGENTS.md                    ← Codex용 실행 규칙 (CLAUDE.md와 같은 절차·안전수칙)
├── .mcp.json                    ← korean-law-mcp(법령·판례 조회) 서버 등록
├── .claude/skills/              ← 5단계 절차 정의 (원본)
├── .codex/skills/               ← Codex용 미러 (자동 생성 — 직접 수정 금지)
├── scripts/                     ← sync_codex_mirror.py (미러 재생성 스크립트)
├── cases/                       ← 사건 폴더 모음 (공개본에는 사건 파일 미포함)
│   └── README.md                ← 실제 사건 파일은 로컬에서만 생성·보관
├── rules/                       ← 인용 표기법, 요건사실, 절차비용 등 작성 규칙
└── templates/                   ← 워드 서식 템플릿과 변환 스크립트
```

### ⚠️ 공개 저장소의 `cases/`에는 사건 파일을 넣지 않습니다

이 공개본은 폴더 구조만 유지합니다. 실제 사건을 만들 때는 로컬에서 새 폴더를
만들어 쓰고, 의뢰인 자료·서면·검증보고 등 사건 파일은 공개 저장소에 커밋하지
마세요.

내부 개발본에서 사용하는 테스트 사건과 골든 케이스는 공개본에 포함하지 않았습니다.

## 안전장치 — 이 도구가 스스로 지키는 5가지 규칙

AI가 그럴듯한 거짓을 쓰는 사고를 막기 위해, 에이전트는 다음을 강제로 지킵니다
(자세한 내용은 [CLAUDE.md](CLAUDE.md)):

1. **사실 생성 금지** — 입력 자료에 없는 사실·수치·날짜는 쓰지 않습니다. 모든 사실관계에는 출처(어느 입력 파일, 어느 증거)가 붙습니다.
2. **인용 단일 진입로** — 리서치 파일에 원문이 발췌된 법령·판례만 인용할 수 있습니다.
3. **모르면 "확인필요"** — 이율·관할처럼 불확실한 값은 추측하지 않고 `[변호사 확인 필요]`로 표시합니다.
4. **검증 게이트** — 인용 검증에서 ❌가 하나라도 나오면 그 서면은 "제출 금지"입니다.
5. **렌더 전 점검** — 워드 파일을 만들기 전, 데이터와 서식이 맞는지 기계적으로 검사합니다.

## 자주 묻는 질문

**Q. 슬래시 명령어(`/case-intake` 같은 것)를 꼭 외워야 하나요?**
A. 아니요. "새 사건 등록해줘", "소장 써줘"처럼 자연어로 말해도 에이전트가 알아서 해당 절차를 실행합니다. 슬래시 명령은 정확히 그 단계를 지정하고 싶을 때 쓰는 단축키입니다.

**Q. 나온 소장을 바로 제출해도 되나요?**
A. **안 됩니다.** 모든 산출물은 초안입니다. 검증보고가 전부 ✅여도, 법리 판단과 전략은 변호사 몫입니다. 검증은 "인용이 원문과 일치한다"는 것만 보증합니다.

**Q. 법령이 개정되면 어떻게 하나요?**
A. 별도로 할 일이 없습니다. korean-law-mcp가 국가법령정보센터를 그때그때 조회하므로 항상 최신 법령을 봅니다. 리서치 단계에서 인용하는 법령이 현행인지(언제 개정됐는지)도 자동으로 확인합니다.

**Q. 사건 폴더 이름은 마음대로 지어도 되나요?**
A. `연도_의뢰인_사건요지` 형식(예: `2026_홍길동_임금미지급` 또는 `2026_홍길동 임금 미지급` — 공백도 가능)을 따라주세요. `/case-intake`가 자동으로 이 형식으로 만들어 줍니다.

**Q. 작업 도중 단계를 건너뛰어도 되나요?**
A. 권장하지 않습니다. 리서치 없이 소장을 쓰면 인용할 수 있는 법령·판례가 없어서 초안이 비게 됩니다. 순서대로: 사건 등록 → 리서치 → 초안 → 검증.

**Q. 워드 파일이 안 만들어져요.**
A. 대부분 `docxtpl` 미설치가 원인입니다. `pip install docxtpl`을 실행하고 다시 시도하세요. 그래도 안 되면 에이전트에게 오류 메시지를 그대로 보여주면 됩니다.
